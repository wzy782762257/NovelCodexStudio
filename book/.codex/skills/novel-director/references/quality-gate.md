# 总编质量门

## 硬门

以下任一项失败都禁止提交：

- 与世界规则、能力上限、既有事实冲突
- 时间倒流、倒计时错误、人物同时出现在不可能抵达的地点
- 人物使用其不应知道的信息
- 行为与核心动机冲突且正文没有给出转变依据
- 关键结果没有可见原因，或章纲必须节点缺失
- 上游 prewrite、precommit 或 postcommit gate 失败
- 审查结果存在 blocking issue、pending disambiguation 或 missed node

## 软门

每项按 0-100 评估，低于阈值先定点返工：

| 维度 | 阈值 |
|---|---:|
| 剧情推进 | 75 |
| 人物一致性 | 80 |
| 连贯性 | 80 |
| 节奏 | 72 |
| 章末追读力 | 75 |
| 文风自然度 | 72 |

额外检查：

- 最近三章不能重复相同的冲突结构、开场方式或章末悬念。
- 对话必须改变关系、信息或决策，不能只复述设定。
- 每章至少产生一个不可无损删除的信息增量。
- 避免连续短句、排比式解释、总结式收尾和泛化情绪词堆叠。

## 停机条件

- 同一章三轮返工仍有硬门失败。
- 需要改写已提交章节才能消除冲突。
- 章纲存在互斥要求或缺少决定剧情方向的关键信息。
- 连续两章低于剧情推进阈值。
- 上游数据库、commit 链或投影链健康检查失败且幂等重试无效。
